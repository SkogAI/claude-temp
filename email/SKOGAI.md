# @inbox/ - dumping ground for quick capture

<what_is_this>

quick capture location for thoughts, ideas, links, notes that need processing later.
inbox zero mindset: capture fast, process later, file appropriately.

</what_is_this>

<when_to_use>

- quick idea that needs recording -> create note here
- link to process later -> capture here
- something that doesn't fit elsewhere yet -> inbox

</when_to_use>

<workflow>

1. capture: create yyyy-mm-dd-topic.md or append to today's file
2. process: periodically review inbox
3. file: move processed items to knowledge/, memory/, or projects/
4. clear: delete or archive processed items

</workflow>

<file_naming>

- daily capture: 2026-01-19.md (multiple items in one file)
- specific capture: 2026-01-19-idea-name.md (single focused item)

</file_naming>
